Grammatica
Tap "+ collegamento" → popover ancorato al bottone.
Search input 48px (HeroUI <Input size="lg">) con focus auto.
Lista raggruppata per TipoLink (parentela, adempimento, parallelo, ...): label gruppo = TipoLink esplicito, riga 48px = elemento target.
Tap riga "Isacco" (parentela) → row inline Ruolo appare con chip 44px (figlio attivo di default per maschi).
Tap fuori → commit (coerente con sketch 1A).
2 tap totali per il caso comune (riga elemento + ruolo) — il più veloce
Ruolo appare solo per parentela — zero rumore per altri TipoLink
Search filtra tutti i gruppi insieme — 1 query, N risultati
Group label espliciti dissolvono l'ambiguità "TipoLink implicito"
Touch target 48px su row + 44px su chip ruolo — iPad-friendly
Su workspace con >100 elementi serve sempre la search (lista lunga senza filtri)
Cosa succede se Isacco fosse compatibile con due TipoLink? (es. parentela + parallelo) — appare due volte? Decidere

Note iPad-specific
  • Popover locked: HeroUI <Popover> è la primitiva — questo sketch confronta solo la grammatica interna. Tutte e 3 le alternative anchored al "+ collegamento" button con offset 8px.
  • Larghezza popover: 380px = comodo su detail pane 520px senza coprirlo del tutto. Su iPad portrait si adatta auto a max-w viewport-50.
  • Touch target: tutti i listbox row 48px, chip 44px — Apple HIG e WCAG 2.5.5 compliant.
  • Link bidirezionali: creando "Abraamo →padre→ Isacco" il dominio crea automaticamente "Isacco →figlio→ Abraamo" (vedi features/elemento/rules.ts). Mai mostrare l'inverso nel picker.
  • Coerenza con altri sketch: commit blur-to-save (sketch 1A), chip Ruolo identici per geometria a quelli di sketch 2C/4 (44px tap target, HeroUI <Chip variant="bordered">).
