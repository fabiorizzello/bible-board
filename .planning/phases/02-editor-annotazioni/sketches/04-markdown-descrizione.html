Markdown descrizione — Milkdown

Scenario: Abraamo ha una descrizione markdown con grassetto, link a Isacco, e una lista. Come si edita senza rompere il flusso inline per-campo?

B I S H1 H2 🔗 </>
Grammatica
Tap nel campo → diventa editable con bordo teal (coerente con altri campi).
Nessun chrome visibile finché non selezioni. Seleziona testo → bubble appare sopra con B / I / S / H1 / H2 / link / code.
Shortcut markdown inline (**bold**, - lista) auto-convertiti da Milkdown.
Tap fuori dal campo → commit (blur-to-save, coerente con sketch 1).
+Chrome zero fino a quando serve — massima densità
+Mental model familiare (Notion, Linear docs)
+Coerente con "inline per-campo" — il campo sembra solo testo
−Selezione touch su iPad è fumosa — tap-hold loupe, a volte scomoda
−Features discoverability bassa: l'utente deve sapere "seleziona per formattare"
−Liste e heading richiedono shortcut markdown o bubble H1/H2 (non ovvio)

Grammatica
Tap nel campo → bordo teal + toolbar appare in alto, sticky al campo.
Tutti gli strumenti visibili: format, heading, lista, link, code, quote. Tap cursore in posizione → l'active state riflette il contesto.
Shortcut markdown inline sempre disponibili in parallelo.
Tap fuori → toolbar scompare + commit.
+Massima discoverability — tutti i tool sono lì
+Touch-friendly: non richiede selezione preliminare
+Heading/lista con un tap — zero shortcut da imparare
−Chrome pesante — il campo descrizione diventa visivamente un editor, rompe la sensazione "tutto è solo testo"
−Occupa verticale prezioso sul detail pane (iPad landscape)
−Meno coerente con gli altri campi inline (nome, data, tipo) che non hanno chrome

Grammatica
Tap nel campo → bordo teal, ma il campo sembra identico al view (zero chrome).
Inline: shortcut markdown (**grassetto**, *corsivo*, [link](url)) auto-convertiti live.
Blocchi: digita / → popover menu con lista, heading, quote, code, check.
Tap fuori → commit.
+Chrome assolutamente zero — densità massima, coerenza perfetta con altri campi inline
+Slash menu è popover (stesso primitivo del collegamento picker)
+Power user gode — tutto da tastiera, zero tap su chrome
−Discoverability bassa: senza tastiera, l'utente deve sapere che / esiste
−Grassetto/corsivo richiedono shortcut markdown (no bold button) — rischio per utenti non power
−Keyboard software iPad deve essere aperta per digitare / — se l'utente apre con pencil è fottuto
Note:
  • Editor locked: Milkdown (MIT, headless, ProseMirror). Lo sketch confronta solo la UI chrome proposta sopra Milkdown.
  • Data model invariato: Elemento.descrizione: string rimane markdown plain text. Milkdown serializza/deserializza al boundary adapter — il dominio non vede MDAST.
  • Commit: tutte le 3 alternative ereditano la decisione di sketch 1 (blur-to-save). Tap fuori dal campo salva; Esc annulla.
  • Coerenza visiva: tutte e 3 usano lo stesso bordo teal + shadow del campo in edit (uguale a sketch 1), per mantenere il "questo è un campo inline" anche quando sotto c'è un editor markdown.
  • Nessuna alternativa usa un modal o un editor full-screen separato — violerebbe il principio "nessun mode swap".
