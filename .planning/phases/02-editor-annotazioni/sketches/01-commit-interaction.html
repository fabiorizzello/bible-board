Commit & touch interaction

Come un campo entra/esce da edit mode su iPad e quando le modifiche si salvano.

A

Blur-to-save

stile Notion / Apple Notes
personaggio
Abraamo
Nascita 2000 a.E.V.
Morte 1825
Tribù Ebrei
Ruoli patriarca, profeta
Grammatica
Tap sul row → edit
Tap fuori (altro campo o area vuota) → save
Esc → annulla · ↵ → save & next
+Zero chrome — densità massima, Notion feel
+Un solo tap per editare, nessun pulsante
−"Tap fuori" ambiguo su iPad portrait
−Nessun cue esplicito di "hai salvato"
−Rischio commit accidentali (tap per sbaglio)
B

Pencil + Enter/Esc

stile Linear — keyboard-first
personaggio
Abraamo
Nascita 2000 a.E.V.
Morte 1825
↵ salva · Esc annulla
Tribù Ebrei
Ruoli patriarca, profeta
Grammatica
Tap sul row o sulla pencil → edit
↵ Enter → save · Esc → cancel
Pencil sempre visibile su iPad (no hover)
+Affordance esplicita, discoverable
+Keyboard shortcut chiari (Enter/Esc)
+No commit accidentali — devi premere Enter
−Pencil icons = più chrome visivo
−Su iPad senza tastiera "Enter" non esiste nativamente — keyboard software ha ↵
C

Save/Cancel inline

explicit commit buttons
personaggio
Abraamo
Nascita 2000 a.E.V.
Morte 1825
Tribù Ebrei
Ruoli patriarca, profeta
Grammatica
Tap sul row → edit
Tap ✓ → save · Tap ✕ → cancel
↵ = ✓ · Esc = ✕
+Commit esplicito — zero ambiguità
+Touch-first, no keyboard needed
+Undo ovvio (tap ✕)
−Meno minimale — 2 button chrome per ogni edit
−Due tap per azione (edit + save)
−Feel più "form tradizionale" che "app nativa"
Note:
  • Il paradigma inline per-campo è già deciso (editingFieldId sostituisce isEditing). Qui scegli solo come funziona il singolo campo in edit mode.
  • Le alternative A e C si auto-estendono ai campi multi-line (descrizione Milkdown): A usa blur, C usa ✓/✕ sotto l'editor.
  • Su iPad senza tastiera fisica, Enter/Esc richiedono la tastiera software — pensa se è accettabile.
